# Практические работы №7–11

Цикл практических заданий по разработке fullstack-приложения с использованием Node.js и React.js. В ходе работ реализованы: REST API, аутентификация с JWT-токенами, ролевая модель доступа, а также клиентская часть.

## Практическая работа №7 — REST API на Node.js

Создано серверное приложение на Node.js с базовыми маршрутами для управления пользователями и товарами.

### Маршруты

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/auth/register` | POST | Регистрация пользователя |
| `/api/auth/login` | POST | Вход в систему |
| `/api/products` | POST | Создать товар |
| `/api/products` | GET | Получить список товаров |
| `/api/products/:id` | GET | Получить товар по id |
| `/api/products/:id` | PUT | Обновить параметры товара |
| `/api/products/:id` | DELETE | Удалить товар |

### Сущности

**Пользователь:**
- `id`, `email`, `first_name`, `last_name`, `password` (хешируется; email используется как логин)

**Товар:**
- `id`, `title`, `category`, `description`, `price`

## Практическая работа №9 — Refresh-токены

Доработано приложение из практической работы №8: добавлена генерация refresh-токенов и маршрут для их обновления.

### Новые маршруты

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/auth/refresh` | POST | Обновление пары access/refresh токенов |

### Формат ответа `POST /api/auth/login`

```json
{
  "accessToken": "<access_token>",
  "refreshToken": "<refresh_token>"
}
```

Маршрут `POST /api/auth/refresh` принимает refresh-токен из заголовков и возвращает новую пару токенов в том же формате.

## Практическая работа №11 — Ролевая модель доступа

| `/api/users` | GET | Администратор | Получить список пользователей |
| `/api/users/:id` | GET | Администратор | Получить пользователя по id |
| `/api/users/:id` | PUT | Администратор | Обновить информацию пользователя |
| `/api/users/:id` | DELETE | Администратор | Заблокировать пользователя |
| `/api/products` | POST | Продавец | Создать товар |
| `/api/products` | GET | Пользователь | Получить список товаров |
| `/api/products/:id` | GET | Пользователь | Получить товар по id |
| `/api/products/:id` | PUT | Продавец | Обновить параметры товара |
| `/api/products/:id` | DELETE | Администратор | Удалить товар |

### Возможности клиентской части

**Товары:**
- Просмотр списка, просмотр по id, создание, редактирование, удаление

**Пользователи (только для Администратора):**

## Практическая работа №13 — Базовый офлайн-функционал

Разработка клиентской части веб-приложения для управления списком дел с обеспечением базовой работы без подключения к сети.

### Реализованный функционал

- Просмотр списка задач/заметок.
- Добавление новых задач и их локальное сохранение в localStorage.
- Регистрация Service Worker для кэширования статических ресурсов.
- Обработка отсутствия сети: страница загружается из кэша, а интерфейс просмотра и добавления задач остается полностью работоспособным в офлайн-режиме.

## Практическая работа №16 — WebSockets и Push-уведомления
Реализация серверной части на Node.js и внедрение двусторонней связи в реальном времени, а также системы подписок на push-уведомления.
### Новые маршруты API
| Маршрут | Метод | Описание |
|---|---|---|
| `/subscribe` | POST | Сохранение подписки пользователя на уведомления |
| `/unsubscribe` | POST| Удаление подписки пользователя |

### События Socket.IO
| Событие | Направление | Описание |
|---|---|---|
| newTask | Клиент → Сервер | Отправляется клиентом при добавлении новой задачи |
| taskAdded | Сервер → Клиент | Рассылается сервером для показа всплывающего сообщения всем подключенным клиентам |
### Реализованный функционал
- Серверная часть с генерацией собственных VAPID-ключей.
- Интеграция Socket.IO для мгновенной синхронизации данных.
- Реализация подписки/отписки на push-уведомления через PushManager.
- UI-интерфейс (кнопки) для управления состоянием уведомлений.
## Практическая работа №17 — Напоминания и интерактивные уведомления
Доработка приложения из практической работы №16: реализация отложенных напоминаний и интерактивных действий в push-уведомлениях.
### Реализованный функционал
- Форма создания заметки расширена полями выбора даты и времени для напоминания.
- Изменена структура хранения в localStorage: добавлены уникальные идентификаторы и поле reminder (timestamp).
- На сервере реализован планировщик push-уведомлений (с использованием setTimeout) и хранение активных таймеров.
- В Service Worker добавлена обработка действий из уведомлений (Action Buttons): Обработка нажатия на кнопку «Отложить на 5 минут». Отправка запроса к серверу для переноса таймера напоминания.
- Гарантирована работоспособность напоминаний в фоновом режиме, даже если вкладка с приложением закрыта.
